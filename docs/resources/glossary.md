# Glossary

A comprehensive glossary of terms used throughout the TechMaster Learning Notes.

## A

**API (Application Programming Interface)**
: A set of protocols and tools for building software applications. Defines how software components should interact.

**Auto Scaling**
: Automatically adjusting compute resources based on demand to maintain performance and optimize costs.

**Availability Zone (AZ)**
: Isolated locations within a cloud region that provide redundancy and fault tolerance.

## B

**Bucket**
: A container for storing objects in cloud storage services like Amazon S3.

**Blue/Green Deployment**
: A deployment strategy that reduces downtime by running two identical production environments.

## C

**CDN (Content Delivery Network)**
: A distributed network of servers that delivers content to users based on their geographic location.

**CI/CD (Continuous Integration/Continuous Deployment)**
: Practices that enable frequent code integration and automated deployment to production.

**Container**
: A lightweight, standalone package that includes everything needed to run an application.

**CRUD (Create, Read, Update, Delete)**
: The four basic operations for persistent storage.

## D

**Docker**
: A platform for developing, shipping, and running applications in containers.

**DynamoDB**
: Amazon's fully managed NoSQL database service.

## E

**EC2 (Elastic Compute Cloud)**
: Amazon's web service that provides resizable compute capacity in the cloud.

**ECS (Elastic Container Service)**
: Amazon's container orchestration service for Docker containers.

**Encryption**
: The process of encoding data to prevent unauthorized access.

## F

**Failover**
: The automatic switching to a redundant system when the primary system fails.

**Function as a Service (FaaS)**
: A serverless computing model where code runs in response to events.

## I

**IAM (Identity and Access Management)**
: Service for securely controlling access to cloud resources.

**IaC (Infrastructure as Code)**
: Managing and provisioning infrastructure through code rather than manual processes.

**Instance**
: A virtual server in the cloud.

## K

**Kubernetes (K8s)**
: An open-source container orchestration platform.

**KMS (Key Management Service)**
: Service for creating and managing cryptographic keys.

## L

**Lambda**
: Amazon's serverless compute service that runs code in response to events.

**Load Balancer**
: Distributes incoming traffic across multiple targets to ensure availability and reliability.

## M

**Microservices**
: An architectural style that structures an application as a collection of loosely coupled services.

**Multi-AZ**
: Deployment across multiple Availability Zones for high availability.

## N

**NAT (Network Address Translation)**
: Remapping IP addresses by modifying network address information.

**NoSQL**
: Non-relational database systems designed for distributed data stores.

## O

**Object Storage**
: Storage architecture that manages data as objects rather than files or blocks.

**Orchestration**
: Automated configuration, coordination, and management of computer systems and software.

## P

**Pod**
: The smallest deployable unit in Kubernetes, containing one or more containers.

**Private Subnet**
: A subnet with no direct route to the internet gateway.

**Public Subnet**
: A subnet with a route to an internet gateway.

## R

**RDS (Relational Database Service)**
: Amazon's managed relational database service.

**Region**
: A geographic area containing multiple Availability Zones.

**Replication**
: Copying data across multiple locations for redundancy and availability.

## S

**S3 (Simple Storage Service)**
: Amazon's object storage service.

**Scalability**
: The ability to handle increased load by adding resources.

**Serverless**
: Cloud computing model where the cloud provider manages infrastructure.

**Subnet**
: A logical subdivision of an IP network.

## T

**Terraform**
: An infrastructure as code tool for building, changing, and versioning infrastructure.

**Throughput**
: The amount of data processed in a given amount of time.

## V

**VPC (Virtual Private Cloud)**
: A logically isolated section of the cloud where you can launch resources.

**VPN (Virtual Private Network)**
: Encrypted connection over the internet between networks.
