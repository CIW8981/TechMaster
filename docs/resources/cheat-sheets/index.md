# Cheat Sheets

Quick reference guides for various technologies and services covered in TechMaster Learning Notes.

## Available Cheat Sheets

### AWS Services

#### Compute
- **EC2**: Instance types, pricing models, and common commands
- **Lambda**: Function configuration, triggers, and best practices
- **ECS/EKS**: Container service commands and configurations

#### Storage
- **S3**: Bucket operations, storage classes, and CLI commands
- **EBS**: Volume types, snapshots, and performance characteristics
- **EFS**: File system operations and mount configurations

#### Networking
- **VPC**: Subnet configuration, routing, and security groups
- **Route 53**: DNS record types and routing policies
- **CloudFront**: Distribution configuration and cache behaviors

#### Databases
- **RDS**: Database engines, backup strategies, and scaling options
- **DynamoDB**: Table design, query patterns, and capacity modes
- **ElastiCache**: Redis and Memcached configurations

#### Security
- **IAM**: Policy syntax, roles, and best practices
- **KMS**: Key management and encryption operations
- **Secrets Manager**: Secret rotation and retrieval

### Container Technologies

#### Docker
- **Commands**: Build, run, and manage containers
- **Dockerfile**: Best practices and common instructions
- **Docker Compose**: Multi-container application definitions

#### Kubernetes
- **kubectl Commands**: Resource management and debugging
- **YAML Manifests**: Pod, Service, and Deployment configurations
- **Troubleshooting**: Common issues and solutions

### Infrastructure as Code

#### Terraform
- **Commands**: Init, plan, apply, and destroy workflows
- **Configuration**: Resource syntax and module usage
- **State Management**: Remote state and locking

### Programming

#### Python
- **AWS Boto3**: Common SDK operations for AWS services
- **Virtual Environments**: Setup and dependency management
- **Common Patterns**: Error handling and best practices

#### JavaScript/Node.js
- **AWS SDK**: Service client usage and async patterns
- **Package Management**: npm and yarn commands
- **Common Patterns**: Promises, async/await, and error handling

## How to Use These Cheat Sheets

1. **Quick Reference**: Use these sheets for rapid lookup of commands and syntax
2. **Study Aid**: Review before exams or interviews
3. **Development**: Keep handy while coding or configuring services
4. **Troubleshooting**: Reference when debugging issues
